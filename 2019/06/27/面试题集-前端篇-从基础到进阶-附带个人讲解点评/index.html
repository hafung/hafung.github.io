JavaScript的8种数据类型

  1. 基本数据类型

  • undefined: 代表已经声明了但未被赋值的变量或者本该有值但因为程序出错没有被赋值。

    注意：typeof(undefined) 返回也是 ‘undefined’。undefined在某些浏览器不会被作为特殊的保留字处理，因此可能会被篡改。

  • null: 可以简单理解为空对象。

    注意：typeof(null)返回Object，但null并非Object，具有null值的变量也并非object。算是特殊对象

  • Boolean: 布尔类型，true or false，是就是，非就非，没有疑义。对就对，错就错，绝对明确。既能被代码处理，也可以控制代码的流程。

  • Number: 线性的事物，大小和次序分明，多而不乱。便于代码进行批量处理，也控制代码的迭代和循环等。

    注意：typeof(NaN)和typeof(Infinity)都返回number 。NaN参与任何数值计算的结构都是NaN，而且 NaN != NaN 。Infinity / Infinity = NaN 。

  • String: 字符串类型，面向人类的理性事物，而不是机器信号。人机信息沟通，代码据此理解人的意图等等，都靠它了。

  1. 引用数据类型：Object、Array、Function

ajax请求的原理

通过XMLHttpRequest对象来向服务器发送异步请求，从服务器获取数据；然后用JavaScript来操作DOM而更新页面

关键词：异步，http请求，客户端发起，回调，…

简单的说，也就是JavaScript可以及时的向服务器提出请求并及时响应。而不阻塞用户。达到无刷新效果。

由事件触发，创建一个XMLHttpRequest对象，把HTTP方法 （POST/GET）和目标URL以及请求返回后的回调函数设置到XMLHttpRequest对象，通过XMLHttpRequest向服务器发送请求，请求发送后继续响应用户的界面交互，只有等到请求真正从服务器返回的时候才调用callback()函数，对响应数据进行处理。

关于跨域

跨域本质是浏览器的为了安全起见的一种限制，不符合同源策略的都算跨域，服务端的请求不存在跨域…

同源策略指的是：协议，域名，端口相同，同源策略是一种安全协议。指一段脚本只能读取来自同一来源的窗口和文档的属性。

常用解决方案：

  1. jsonp

    原理是动态插入script标签，只能模拟get请求, 需要服务端配合

  2. CORS

    需要后端配合，前端无需作特殊处理

  3. nginx反向代理-转发请求

    需要后端人员\运维配合

闭包（Closure）

闭包的本质是创建块级作用域，保留变量在内存中不被回收(因为用某些方法一些引用了这个变量)；然后es6的let语法出来后，闭包在js前端的场景已经很少了

原型链\原型

每个JS对象都有一个prototype原型属性，指向该对象继承的原型。

原型对象上有一个 constructor 属性，该属性指向的就是构造函数。

而实例对象上有一个 __proto__ 属性，该属性也指向原型对象，并且该属性不是标准属性，不可以用在编程中，该属性用于浏览器内部使用。

原型链：其实就是有限的实例对象和原型之间组成有限链，就是用来实现共享属性和继承的。

JavaScript的继承同其他面向对象的继承是由差异的，js本质是用对象模拟的类，用原型的继承模拟类的继承，以及其他面向对象的一些方法\特征；es6的class也是语法糖，本质也是prototype

关于this

this总是指向一个对象，具体是运行时基于函数的执行环境动态绑定的，而非函数被声明时的环境。

普通函数调用时，一般指向window对象；

对象方法调用时，指向该对象；

也就是用new去调用一个方法

如果对象方法里有局部方法，里面的this会指向window对象；

用es6的箭头函数可以维持执行上下文不变，也就是保持this的指向

在ES5模式下，this指向window的都会变成undefined；

Apply，call时，动态改变this

Array原型上的一些常用方法

concat() 连接两个或更多数组

返回新数组，注意该方法只是浅拷贝

splice(index,len,[item]) 删除元素，并向数组添加一个新元素。

slice() 从某个已有的数组返回选定的元素

join() 把数组的所有元素放入一个字符串，元素通过指定的分隔符进行分割

push() 在数组后添加元素，并返回新的长度

unshift() 在数组最前添加元素

pop() 删除数组最后一个元素并返回该元素的值

reverse() 颠倒数组中元素的顺序

shift() 删除并返回数组中第一个元素

sort() 对数组元素进行排序

toSource() 返回该对象的源代码

toString() 把数组转换为字符串并返回结果

将数组转为字符串；不止Array原型上有该方法
console.log([1, 2, 'a', '1a'].toString()); // expected output: "1,2,a,1a"

toLocaleString() 把数组转换为本地数组，并返回结果

不止Array原型上有该方法；Number, Object, Date…
一般也是用于转换数组里的字符串格式的时间的元素

[1, 'a', new Date('21 Dec 1997 14:12:00 UTC')].toLocaleString('en', {timeZone: "UTC"}); // // expected output: "1,a,12/21/1997, 2:12:00 PM",

valueOf() 返回对象的原始值

map(callback(item, index, array)=>{}) 创建一个新数组，其结果是该数组中的每个元素都调用一个提供的函数后返回的结果。

foreach(callback(item, index, array)=>{}) 遍历数组，对数组的每个元素执行一次提供的函数; 没有返回

无法return，这点不同于for循环，因为是回调，即使return了也会完整的遍历；与map的不同是，不返回新数组，一般用于对当前数组的修改

filter(callback(item, index, array)=>{}) 创建一个新数组, 其包含通过所提供函数实现的测试的所有元素。

find(callback(item, index, array)=>{}) 返回数组中满足提供的测试函数的第一个元素的值。否则返回 undefined。

findIndex() 返回数组中找到的元素的索引，而不是其值

every() 测试一个数组内的所有元素是否都能通过某个指定函数的测试。它返回一个布尔值。若收到一个空数组，此方法在一切情况下都会返回 true。

some() 是否至少有一个元素可以通过被提供的函数方法。该方法返回一个Boolean类型的值。对于空数组上的任何条件，此方法返回false。

includes() 用来判断一个数组是否包含一个指定的值，根据情况，如果包含则返回 true，否则返回false。对象数组不能使用includes方法来检测

建议用来代理对数组的indexOf操作，如果只是判断有无包含的话

具体用法，可以多查阅MDN来熟悉

日常处理业务逻辑很大一部分是对数组的处理，而对数组的处理中，活用这些方法会事半功倍

前端性能优化方案

本质是减少CPU和GPU的计算或者同等条件\实现下减少内存的使用；基于目前硬件的性能过剩以及优越的网络环境，只要不是死循环或者内存泄漏，且多加载个几百K的js库一般是无感知的，所以一般性能的瓶颈不在前端；前端的优化更多是锦上添花而已

当然如果项目很庞大了，各种依赖，比如现在流行的SPA应用

优化方向

  1. 减少页面体积，提升网络加载

    • 静态资源的压缩合并（JS 代码压缩合并、CSS 代码压缩合并、雪碧图）

    • 静态资源缓存（资源名称加 MD5 戳）

    • 使用 CDN 让资源加载更快

  2. 优化页面渲染

    • CSS 放前面，JS 放后面

      老生常谈，现在都是自动化构建了，一般不会有这种问题

    • 懒加载（图片懒加载、下拉加载更多）

    • 减少DOM 查询，对 DOM 查询做缓存; 减少DOM 操作，多个操作尽量合并在一起执行（DocumentFragment）

      现在主流都用mvvm框架了，基本避免了这个问题，因为很少会用DOM操作了

    • 事件节流

    • 尽早执行操作 - DOMContentLoaded

      jQuery 中经常使用的 $(document).ready(function() { // ...代码... }); 其实监听的就是 DOMContentLoaded 事件，页面被解析后，我们就可以访问整个页面的所有dom元素，可以缩短页面的可交互时间，提高整个页面的体验; 传统的window.onload是需要页面完整加载后才触发，包括js，图片等资源；不过基于目前流行的mvvm框架是有对应的钩子函数去处理的

    • 使用 SSR 后端渲染，数据直接输出到 HTML 中，减少浏览器使用 JS 模板渲染页面 HTML 的时间

      个人认为SSR目前只是过渡方案
